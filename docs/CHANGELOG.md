## 문제해결 기록

### [PROB-001] window.storage 저장 실패 (2026-04-03)
**문제**: JSX 아티팩트에서 기록 저장 시 "저장 실패" 메시지
**원인**: Claude 아티팩트 환경에서 `window.storage` API 불안정
**해결**: localStorage로 전환 + 로컬 Vite 환경에서 실행

### [PROB-002] 통계 일수 오분류 (2026-04-04)
**문제**: 가스참(value:5), 잔변감(value:6) 추가 후 "심함"으로 잘못 집계됨
**원인**: 숫자값 크기로 worst 비교하여 5, 6이 2(아랫배아픔)보다 크게 판단됨
**해결**: 각 증상에 `sev` 필드 추가하여 severity 기준으로 비교하도록 변경

### [PROB-003] 헤더 코드 중복 삽입 (2026-04-04)
**문제**: 내보내기/가져오기 버튼 추가 중 JSX 중복 삽입으로 흰 화면
**원인**: str_replace 특수문자 처리 실패로 코드 이중 삽입
**해결**: App.jsx 전체 파일 교체로 해결

### [PROB-004] 어제 저녁 자동입력 미작동 (2026-04-05)
**문제**: 페이지 새로고침 시 어제 저녁 필드가 항상 비어있음
**원인**: `useState(mkForm())` — 빈 배열로 mkForm 호출하여 localStorage 데이터 무시
**해결**: lazy initializer로 localStorage 직접 읽어 `mkForm(saved)` 호출로 변경

### [PROB-005] 복합 상태가 통계에서 심함으로 집계 (2026-04-05)
**문제**: 히스토리 카드는 🟡 복합인데 통계는 심함으로 표시
**원인**: `worstSev()`가 severity 최댓값만 반환하여 복합 판별 불가
**해결**: `isMixed()` 헬퍼 추가, badDays에서 복합일 제외하고 mixedDays로 별도 집계
